### FIX 2 (CRITICAL): Update Visual S&OP Chart Capacity Line

**File:** `templates/student/module2.html`

**Find these lines (around line 853–857) inside the `<script type="text/babel">` block:**
```javascript
const capacities = {
    'standard': 30000 * 7,  // Weekly
    'overtime': 38000 * 7,
    'two_shift': 48000 * 7
};
```

**Replace with:**
```javascript
const capacities = {
    'standard': 800 * 7,    // 5,600/week
    'overtime': 1050 * 7,   // 7,350/week
    'two_shift': 1500 * 7   // 10,500/week
};
```

**Why this matters:** The chart draws a red dashed "Capacity Limit" line. With the old values (210,000/week for Standard), the line was way above any realistic production bar, making the chart useless. With the corrected values (5,600/week), the line sits right where students need to see it — showing whether their plan fits within capacity.

### FIX 3 (OPTIONAL): Improve Chart Y-Axis Scale

With the old 30k×7 = 210,000 capacity line, the Y-axis auto-scaled to show 0–250,000, making production bars invisible. With the fix above, the Y-axis should auto-scale to ~0–12,000, which is correct.

If the Y-axis still looks wrong after Fix 2, you can add an explicit domain:

**In the chart `<YAxis>` component, add a domain:**
```jsx
<YAxis 
    label={{ value: 'Units per Week', angle: -90, position: 'insideLeft' }}
    domain={[0, 12000]}
/>
```

This forces the chart to show 0–12,000 on the Y-axis regardless of data, ensuring production bars (~5,000–7,000) and the capacity line (~5,600–10,500) are both visible.
