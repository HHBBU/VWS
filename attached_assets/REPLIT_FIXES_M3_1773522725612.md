## Issues Fixed in Polished Docs (Not Replit)

These were mismatches between the original guide and the engine that I corrected in the polished docs:

### 1. Justification Scoring: "Words" vs "Characters"

**Original guide said:** "≥400 words → 3 pts"
**Engine does:** `len(justification) >= 400` (characters, not words)
**Polished guide now says:** "≥400 characters (~60+ words) → 3 pts"

Practical impact is low (any real paragraph clears 400 characters), but the unit was wrong.

### 2. Missing Cost Constants

**Original guide:** Did not list holding cost, stockout penalty, or carbon tax
**Engine uses:** €0.10/unit/day holding, €8.00/unit stockout, €0.05/kg carbon
**Polished guide now includes:** Full cost parameter table (Section 7)

Students need these to understand why their total cost is what it is.

### 3. Missing Letter Grade Cutoffs

**Original guide:** No letter grade mapping
**Engine uses:** A≥50, B≥44, C≥38, else D
**Polished guide now includes:** Letter grade table in rubric section

### 4. M2 → M3 Integration Not Mentioned

**Original guide:** No mention that M2 service level affects M3
**Engine does:** `lead_time_stretch = 1.0 + (1.0 - m2_service_level) * 2.0` — low M2 service stretches max lead time by up to 40%
**Polished guide now includes:** Section 2 explaining M2 integration + note in network table about lead time stretch

### 5. Inventory Logic Scoring Details

**Original guide:** Described loosely as "ROP reasonable, Q reasonable"
**Engine does:** Specific deductions: ROP < ideal_min × 0.5 → −4, Q < ideal_min × 0.5 → −3, Q > ideal_max × 2 → −2
**Polished instructor guide now includes:** Full deduction logic with example calculations for typical demand levels

### 6. Network Design Scoring Details

**Original guide:** "Network-service alignment"
**Engine does:** Specific fill rate + network combinations determine points (hybrid+90%=5, centralized+90%=4, centralized+<85%=2, else=3)
**Polished instructor guide now includes:** Exact scoring table
